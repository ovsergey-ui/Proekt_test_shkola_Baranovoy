# Что улучшать дальше

## Качество генерации

- Перейти от простых `{{key}}` и `[key]` к правилам по секциям (заголовки, таблицы, блоки).
- Добавить предпросмотр diff: что именно заменится перед сохранением.
- Поддержать вложенные ключи вида `user.name`, `company.address.city`.

## Анализатор

- Выделение сущностей: даты, суммы, ФИО, номера договоров.
- Более умный скоринг шаблонов (веса по структуре, повторяемости и плотности полей).
- Автогруппировка документов по типам: договор, акт, отчет, письмо.

## Работа с большими объемами

- Вынести сканирование и парсинг в Web Worker для полностью отзывчивого UI.
- Кэшировать результаты в `IndexedDB` для ускорения повторного анализа.
- Добавить инкрементальный анализ: пересчитывать только изменившиеся файлы.

## Форматы

- Улучшить обработку PDF (структурные блоки и таблицы, а не только raw text).
- Добавить поддержку `xlsx` и `csv` как шаблонных источников.
- Поддержать экспорт в несколько форматов (`txt`, `md`, `docx`).

## UX

- Экран "Шаблоны" с карточками и быстрым поиском.
- Мастер генерации в 3 шага: шаблон -> данные -> превью.
- История операций: что анализировали и что генерировали (с timestamp).

## Надежность

- Валидатор JSON-данных с понятными подсказками по ошибкам.
- Более детальные сообщения о permission-ошибках.
- Лог пропущенных файлов: какие не удалось прочитать и почему.

## Продуктовые фичи

- Пакетная генерация нескольких документов за один запуск.
- Пользовательские наборы полей (профили данных).
- Сохранение и загрузка шаблонных правил локально (`localStorage` / `IndexedDB`).

## Рекомендуемый следующий этап (топ-3)

1. Предпросмотр замен перед сохранением.
2. Web Worker для анализа.
3. История генераций.
