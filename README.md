# Локальный Анализатор Документов

Полностью оффлайн веб-приложение на чистом HTML/JS/CSS.  
Работает локально в браузере, без серверов, API и интернета.

## Быстрый старт

1. Убедитесь, что в папке проекта есть `index.html`.
2. Если переносите проект вручную, сохраните основной файл именно как `index.html`.
3. Откройте `index.html` в Chrome или Edge (86+).
4. Нажмите **Выбрать папку**.
5. В системном окне браузера нажмите **Разрешить доступ** к выбранной директории.
6. Нажмите **Анализировать**.
7. Выберите шаблон, введите JSON-данные и нажмите **Генерировать**.

## Структура файлов

```text
Proekt_test_shkola_Baranovoy/
  index.html
  libs/
    pdf.min.js
    pdf.worker.min.js
    mammoth.browser.min.js
```

## Оффлайн библиотеки

Для локального парсинга форматов используются:

- `libs/pdf.min.js`
- `libs/pdf.worker.min.js`
- `libs/mammoth.browser.min.js`

Эти файлы уже скачаны и подключаются локально, поэтому интернет не требуется.

## Мини-тест после запуска

1. Нажмите **Выбрать папку** и выберите директорию с `txt/md/pdf/docx`.
2. Убедитесь, что появился список файлов и счетчик.
3. Нажмите **Анализировать** и проверьте блок `Результат анализа`.
4. Выберите шаблон и сгенерируйте новый документ.
5. Убедитесь, что открылся диалог сохранения и файл сохранился.

## Проверка полностью оффлайн

1. Отключите интернет.
2. Перезагрузите страницу с `index.html`.
3. Проверьте статус парсеров в интерфейсе:
   - `PDF: OK`
   - `DOCX: OK`

Если оба статуса `OK`, приложение работает полностью оффлайн.

## Ограничения текущей версии

- PDF извлекается как текст (без точного восстановления сложных таблиц).
- DOCX читается как raw text (базовый режим).
- Очень большие папки могут сканироваться заметно дольше, но UI остается отзывчивым за счет порционной отрисовки.
